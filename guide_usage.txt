================================================================
AreSearch 使い方ガイド
================================================================

このファイルは、検索の書き方と検索結果の扱い方を扱う。
導入手順は guide_setup.txt、reindex や同期運用は guide_operations.txt を参照する。


# 1. 基本の検索
================================================================

## 1-1. 単一 target 検索

通常は index target 経由で検索する。

    article_index = Article.are_search_index_target(:default)

    result = article_index.are_search_es_search("検索ワード", fields: [:title, :body])

直接呼ぶ場合は SingleSearch を使う。

    result = AreSearch::SingleSearch.search(article_index, "検索ワード", fields: [:title, :body])

検索語が nil、空文字、blank の場合、検索語による絞り込みは行わない。

戻り値は SearchResult。
実レコードは result.records から取得する。

    result.records.each do |article|
        puts article.title
    end


## 1-2. 複数モデル横断検索

複数モデルをまとめて検索する場合は MultiSearch を使う。

    result = AreSearch::MultiSearch.search(
        [Article.are_search_index_target(:default), Document.are_search_index_target(:default)],
        "検索ワード",
        fields: [:title, :body],
    )

ショートハンドもある。

    result = AreSearch.multi_search(
        [Article.are_search_index_target(:default), Document.are_search_index_target(:default)],
        "検索ワード",
        fields: [:title, :body],
    )

index_targets には AreSearch::IndexTarget を1件以上渡す。
戻り値の SearchResult は単一 target 検索と同じ。


## 1-3. 類似ドキュメント検索

検索語の代わりに起点レコードを渡す。

    article = Article.find(1)

    article_index = Article.are_search_index_target(:default)

    result = AreSearch::MoreLikeThis.search(
        [article_index],
        article,
        article_index,
        fields: [:title, :body],
    )

ショートハンドもある。

    article_index = Article.are_search_index_target(:default)

    result = AreSearch.more_like_this(
        [article_index],
        article,
        article_index,
        fields: [:title, :body],
    )

index_targets には AreSearch::IndexTarget を1件以上渡す。
横断検索もできる。

    article_index = Article.are_search_index_target(:default)
    document_index = Document.are_search_index_target(:default)

    result = AreSearch::MoreLikeThis.search(
        [article_index, document_index],
        article,
        article_index,
        fields: [:title, :body],
    )

fields は Array<Symbol> で指定する。
MoreLikeThis の fields には boost は指定できない。


## 1-4. 生クエリ検索

AreSearch は Elasticsearch の Query DSL を隠すための gem ではありません。
よく使う検索には are_search_es_search を用意していますが、複雑な検索は RawSearch で Elasticsearch の body を直接書く方針です。
アプリ側が Query DSL を理解しておくことで、将来 gem の実装を変更しても検索ロジックを移し替えやすくなります。

AreSearch に慣れてきたら RawSearch を使う。
Elasticsearch に投げる body を直接渡す。


    body = {
        query: {
            bool: {
                must: [
                    { match: { title: "Rails" } },
                ],
            },
        },
    }

    result = AreSearch::RawSearch.search(
        [Article.are_search_index_target(:default)],
        body,
    )

RawSearch で使える AreSearch 側のオプションは以下だけ。

    :model_results_where
    :model_includes
    :page
    :per_page

where、fields、sort、highlight などは body に直接書く。

body 内に from / size を書いても、page / per_page から算出した値で上書きされる。
SearchResult のページネーション情報と揃えるため。

RawSearch でも、page / per_page から算出した from / size は通常検索と同じく
max_result_window の範囲に補正される。
単一 target ではその target の are_search_es_index_settings[:max_result_window]、
複数 target では対象 target の最小値を使う。
from + size が max_result_window を超える場合は size を縮める。
from が max_result_window 以上の場合は、from を max_result_window に丸め、size を 0 にする。

一方で、RawSearch は body を直接渡す検索なので、track_total_hits: true は自動指定しない。
正確な total hits が必要な場合は、body に track_total_hits: true を明示する。

    result = AreSearch::RawSearch.search(
        [Article.are_search_index_target(:default)],
        body,
        page:     2,
        per_page: 10,
    )

RawSearch で metric aggregation など buckets を持たない aggregation を使う場合、
result.aggs には入らない。
Elasticsearch の生レスポンスは result.raw_response から取得する。

    result.raw_response["aggregations"]["avg_price"]["value"]

body のキーはシンボルでも文字列でもよい。
内部でシンボルキーへ統一される。

Query DSL の仕様は公式リファレンスを参照する。

https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl.html


# 2. よく使う検索パターン
================================================================

## 2-1. シンプルな全文検索

    result = Article.are_search_index_target(:default).are_search_es_search(
        "Elasticsearch入門",
        fields: [:title, :body],
    )

    result.records.each do |article|
        puts article.title
    end


## 2-2. ページネーション付き検索

    result = Article.are_search_index_target(:default).are_search_es_search(
        "Ruby",
        fields:   [:title, :body],
        page:     params[:page],
        per_page: 20,
    )

    @articles = result.records
    @total    = result.records.total_count


## 2-3. 絞り込み付き検索

    result = Article.are_search_index_target(:default).are_search_es_search(
        "Rails",
        fields: [:title, :body],
        where:  { status: "published" },
    )


## 2-4. 絞り込み + ソート + ページネーション

    result = Article.are_search_index_target(:default).are_search_es_search(
        "Rails",
        fields:   { title: 2.0, body: 1.0 },
        where:    { status: "published" },
        sort:     { updated_at: :desc },
        page:     params[:page],
        per_page: 10,
    )


## 2-5. ハイライト付き検索

    result = Article.are_search_index_target(:default).are_search_es_search(
        "Rails",
        fields:    [:title, :body],
        highlight: { fragment_size: 150 },
    )

    result.records_with_target_names.each do |article, target_name|
        puts article.title

        result.highlights_html(article, target_name).each do |fragment|
            puts fragment
        end
    end


## 2-6. 集計付き検索

    result = Article.are_search_index_target(:default).are_search_es_search(
        "Rails",
        fields: [:title, :body],
        where: { status: "published" },
        aggs:  [
            :status,
            { category: { size: 50 } },
        ],
    )

    result.aggs[:category].each do |name, count|
        puts "#{name}: #{count}件"
    end


## 2-7. 複数モデル横断検索

    result = AreSearch::MultiSearch.search(
        [Article.are_search_index_target(:default), Document.are_search_index_target(:default), Wiki.are_search_index_target(:default)],
        "設計ガイド",
        fields:    [:title, :body],
        page:      params[:page],
        per_page:  20,
        highlight: {},
    )

    result.records_with_target_names.each do |record, target_name|
        puts "#{record.class.name}: #{record.title}"

        result.highlights_html(record, target_name).each do |fragment|
            puts fragment
        end
    end


## 2-8. 類似ドキュメント検索

    article = Article.find(params[:id])

    article_index = Article.are_search_index_target(:default)

    result = AreSearch::MoreLikeThis.search(
        [article_index],
        article,
        article_index,
        fields:        [:title, :body],
        min_term_freq: 1,
        where:         { status: "published" },
        where_not:     { id: article.id },
        per_page:      5,
    )

    @similar_articles = result.records


# 3. オプション一覧
================================================================

are_search_es_search / MultiSearch / MoreLikeThis で使える主なオプション。
MoreLikeThis で使えないものには ※ を付ける。

    オプション             型                         デフォルト
    ---------------------------------------------------------------
    :fields                Array<Symbol> or Hash      なし  必須
    :where                 Hash                       なし
    :where_not             Hash                       なし
    :results_where         Hash                  ※   なし  単一 target 検索のみ
    :model_results_where   Hash<Class, Hash>          なし  MultiSearch / MoreLikeThis
    :aggs                  Array<Symbol or Hash>      なし
    :includes              Array or Hash         ※   なし  単一 target 検索のみ
    :model_includes        Hash<Class, Array>         なし  MultiSearch / MoreLikeThis
    :page                  Integer                    1
    :per_page              Integer                    25
    :sort                  Hash or Array         ※   なし  are_search_es_search / MultiSearch
    :highlight             Hash                       なし
    :should                Array<Hash>                なし
    :minimum_should_match  Integer                    1
    :min_term_freq         Integer                    2     MoreLikeThis のみ
    :min_doc_freq          Integer                    5     MoreLikeThis のみ
    :max_query_terms       Integer                    25    MoreLikeThis のみ
    :min_word_length       Integer                    nil   MoreLikeThis のみ


# 4. fields
================================================================

必須オプション。
検索対象フィールドを指定する。
combined_fields クエリに使うため、text 型のフィールドのみ指定できる。

フィールド名だけ指定する場合。

    Article.are_search_index_target(:default).are_search_es_search(
        "検索ワード",
        fields: [:title, :body],
    )

boost を付ける場合。

    Article.are_search_index_target(:default).are_search_es_search(
        "検索ワード",
        fields: { title: 2.0, body: 1.0 },
    )

boost は 1.0 以上の数値。
MoreLikeThis の fields は Array<Symbol> のみ。
MoreLikeThis の fields には boost は指定できない。


# 5. where
================================================================

絞り込み条件を指定する。
複数条件は AND になる。

単一値は term クエリになる。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        where: { status: "published" },
    )

配列は terms クエリになる。
配列内の値は OR 条件。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        where: { user_id: [1, 2, 3] },
    )

複数キーを指定すると AND 条件。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        where: {
            status:  "published",
            user_id: 1,
        },
    )

Hash を渡すと range クエリになる。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        where: { updated_at: { gte: "2024-01-01" } },
    )

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        where: { price: { gte: 1000, lte: 5000 } },
    )

gte / lte / gt / lt 以外の range キーは AreSearch 側では検証しない。
不正なキーを指定した場合は Elasticsearch 側のエラーになる。


# 6. where_not
================================================================

where と同じ記法で、条件に一致するドキュメントを除外する。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        where_not: { id: article.id },
    )

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        where_not: { status: ["draft", "deleted"] },
    )

Hash を渡した場合は range クエリになり、その範囲を除外する。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        where_not: { updated_at: { lt: "2024-01-01" } },
    )


# 7. results_where / model_results_where
================================================================

results_where、model_results_where の値は加工せず ActiveRecord の where にそのまま渡される。

ES の検索結果として返ってきた ID を DB から取得するときに、追加の ActiveRecord where 条件をかける。
ES インデックスと DB の乖離を吸収したい場合に使う。

単一 target 検索では results_where を使う。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        results_where: { status: "published" },
    )

MultiSearch / MoreLikeThis では model_results_where を使う。

    AreSearch::MultiSearch.search(
        [Article.are_search_index_target(:default), Document.are_search_index_target(:default)],
        nil,
        fields: [:title, :body],
        model_results_where: {
            Article  => { status: "published" },
            Document => { visible: true },
        },
    )

ES でヒットした ID が DB に存在しない場合や、条件に合わない場合は結果から除外される。
model_results_where のキーには、検索対象 models に含まれているモデルだけを指定できる。


# 8. aggs
================================================================

フィールドごとの値の分布を集計する。
keyword 型のフィールドに使う。
旧称は facets。

フィールド名だけ指定する場合。

    result = Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        aggs: [:status, :category],
    )

    result.aggs
    # => { status: { "published" => 10, "draft" => 3 }, category: { "tech" => 8 } }

フィールド名だけを指定した場合、terms aggregation の size には
AreSearch.default_aggs_size が使われる。
デフォルトは 200。

フィールドごとに terms aggregation のオプションを指定する場合。

    result = Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        aggs: [
            :status,
            { category: { size: 50 } },
        ],
    )

この場合、status は AreSearch.default_aggs_size、category は size: 50 で集計される。

Hash で指定した値は terms aggregation のオプションとして渡される。
size 以外の terms aggregation オプションも指定できる。

    result = Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        aggs: [
            { category: { size: 50, order: { _count: :desc } } },
        ],
    )

aggs を指定しない場合、result.aggs は空 Hash になる。

    result.aggs
    # => {}

SearchResult の result.aggs は buckets を持つ aggregation だけを簡易形式へ変換する。
RawSearch で avg / sum / min / max など buckets を持たない aggregation を使った場合、
その aggregation は result.aggs では無視される。
必要な場合は result.raw_response で Elasticsearch の生レスポンスを参照する。

    result.raw_response["aggregations"]


# 9. includes / model_includes
================================================================

includes、model_includes の値は加工せず ActiveRecord の includes にそのまま渡される。

DB からレコードを取得するときに ActiveRecord の includes を適用する。

単一 target 検索では includes を使う。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        includes: [:user, :tags],
    )

MultiSearch / MoreLikeThis では model_includes を使う。

    AreSearch::MultiSearch.search(
        [Article.are_search_index_target(:default), Document.are_search_index_target(:default)],
        nil,
        fields: [:title, :body],
        model_includes: {
            Article  => [:user, :tags],
            Document => [:author],
        },
    )

model_includes のキーには、検索対象 models に含まれているモデルだけを指定できる。


# 10. page / per_page
================================================================

ページネーションを指定する。

    result = Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        page:     2,
        per_page: 10,
    )

検索結果のレコードは result.records から取得する。

    result.records.each do |article|
        puts article.title
    end


Kaminari の paginate ヘルパーにもそのまま渡せる。

    paginate result.records

件数やページ情報も result.records から取得できる。

    result.records.current_page
    result.records.per_page
    result.records.total_count
    result.records.es_total_count
    result.records.total_pages

通常検索の page / per_page から作る from / size は、
各 index target の are_search_es_index_settings[:max_result_window] の範囲に補正される。
単一 target 検索ではその target の値を使う。
MultiSearch / MoreLikeThis では、対象 target の値の最小値を使う。

from + size が max_result_window を超える最後のページでは、取得件数が per_page より少なくなることがある。
from が max_result_window 以上の場合は、Elasticsearch へ from: max_result_window, size: 0 を渡す。

total_count は max_result_window を上限に丸めない。
result.records.total_count は、Elasticsearch が返した hits.total.value から、
現在ページで DB 復元できなかった hit 数を差し引いた表示用の件数になる。
Elasticsearch が返した元の hits.total.value は result.records.es_total_count から取得できる。

この補正は、現在ページで観測できた欠落だけを反映する。
全ページを DB 側で再検証するものではないため、表示可能な総件数を厳密に保証するものではない。


# 11. sort
================================================================

検索結果のソート順を指定する。
are_search_es_search と MultiSearch で使える。
MoreLikeThis では使えない。

単一フィールドの場合。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        sort: { updated_at: :desc },
    )

複数フィールドの場合。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        sort: [
            { updated_at: :desc },
            { _score: :desc },
        ],
    )

_score と _doc は予約フィールドとして常に使える。
それ以外は are_search_es_mappings に定義されているフィールドだけ指定できる。

Elasticsearch のソートオプションを渡したい場合は Hash を使う。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        sort: { updated_at: { order: :desc, missing: "_last" } },
    )

Array 形式でも同じ。

    Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        sort: [
            { updated_at: { order: :desc, missing: "_last" } },
            { _score: :desc },
        ],
    )

sort の値の構造は AreSearch 側では検証しない。
不正な値を指定した場合は Elasticsearch 側のエラーになる。


# 12. highlight
================================================================

マッチした箇所を含む断片を取得する。

空 Hash を渡すと、fields と同じフィールドがハイライト対象になる。

    result = Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields:    [:title, :body],
        highlight: {},
    )

対象フィールドを指定する場合。

    result = Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields:    [:title, :body],
        highlight: { fields: [:body] },
    )

フラグメントサイズを指定する場合。

    result = Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        highlight: { fragment_size: 200 },
    )

MoreLikeThis でも使える。

    article_index = Article.are_search_index_target(:default)

    result = AreSearch::MoreLikeThis.search(
        [article_index],
        article,
        article_index,
        fields:    [:title, :body],
        highlight: {},
    )

highlight に指定できるキー。

    キー             型             デフォルト
    -----------------------------------------------------------
    :fields         Array<Symbol>   fields オプションと同じフィールド
    :fragment_size  Integer         Elasticsearch のデフォルト

Elasticsearch から返るハイライトタグは <em>...</em> に固定される。
表示時のタグ変更は SearchResult#highlights_html で行う。


# 13. should / minimum_should_match
================================================================

bool query の should 句を組み立てる。
should にマッチした文書はスコアが加点される。
minimum_should_match と組み合わせると、should を絞り込み条件としても使える。

    result = Article.are_search_index_target(:default).are_search_es_search(
        nil,
        fields: [:title, :body],
        should: [
            { field: :name,   value: "aaa2", boost: 1.0 },
            { field: :name,   value: "aaa3", boost: 2.0 },
            { field: :status, value: ["draft", "review"] },
        ],
        minimum_should_match: 1,
    )

should の各要素は Hash。

    キー        必須    説明
    -----------------------------------------------------------
    :field     必須    対象フィールド名
    :value     必須    マッチさせる値
    :boost     任意    スコアへの加点の重み

value が単一値の場合は term クエリ。

    { field: :status, value: "published" }

value が配列の場合は terms クエリ。

    { field: :status, value: ["draft", "review"] }

value が Hash の場合は range クエリ。

    { field: :price, value: { gte: 1000, lte: 5000 } }

boost は Integer または Float。
0 以上を指定する。
未指定時は 1.0。

minimum_should_match は 0 以上の Integer。
未指定時は 1。
文字列指定には対応しない。

range value 内に boost は指定できない。
boost は should 要素の直下に指定する。

should を指定しなかった場合、minimum_should_match は無視される。


# 14. MoreLikeThis 専用オプション
================================================================

MoreLikeThis では以下の専用オプションを使える。

    :min_term_freq
        同一ドキュメント内で何回以上出現した単語をキーワードに使うか。
        デフォルトは 2。

    :min_doc_freq
        何件以上のドキュメントに出現した単語をキーワードに使うか。
        デフォルトは 5。

    :max_query_terms
        キーワードとして使う単語の最大数。
        デフォルトは 25。

    :min_word_length
        キーワードとして使う単語の最低文字数。
        デフォルトは nil。

例。

    article_index = Article.are_search_index_target(:default)

    result = AreSearch::MoreLikeThis.search(
        [article_index],
        article,
        article_index,
        fields:          [:title, :body],
        min_term_freq:   1,
        min_doc_freq:    2,
        max_query_terms: 20,
        min_word_length: 2,
    )

すべて正の整数で指定する。

# 15. highlights_html
================================================================

マッチ箇所を HTML タグで囲んだフラグメントの配列を返す。

    result.records_with_target_names.each do |article, target_name|
        fragments = result.highlights_html(article, target_name)

        fragments.each do |fragment|
            puts fragment
        end
    end

デフォルトでは <em>...</em> になる。

    result.highlights_html(article, :default)
    # => ["...Railsの<em>ガイド</em>は..."]

タグやclass属性を変更できる。

    result.highlights_html(
        article,
        :default,
        tag:        "mark",
        class_name: "highlight",
    )

    # => ["...<mark class=\"highlight\">Rails</mark>..."]

ヒットなし、または highlight 未指定の場合は空配列を返す。

    []


# 16. highlights_source
================================================================

フラグメントではなく、Elasticsearch の _source を返す。

    source = result.highlights_source(article, :default)

    source[:title]
    source[:body]

戻り値は Hash。
キーは Symbol になる。

ヒットなし、または highlight 未指定の場合は空 Hash を返す。

    {}


# 17. DumpBody
================================================================

DumpBody は、AreSearch の簡易オプションがどのような Elasticsearch body に変換されるかを確認するための機能です。
複雑な検索を RawSearch で書く前に、are_search_es_search / MultiSearch / MoreLikeThis の出力を確認し、Query DSL の形を学ぶ用途にも使えます。

検索を実行せず、組み立てた Elasticsearch body だけを確認したい場合は AreSearch::DumpBody を使う。

    body = Article.are_search_index_target(:default).are_search_es_search(
        AreSearch::DumpBody,
        fields: [:title, :body],
        where:  { status: "published" },
    )

    pp body

are_search_es_search / MultiSearch / MoreLikeThis で使える。
戻り値は SearchResult ではなく Hash。

MultiSearch の例。

    body = AreSearch::MultiSearch.search(
        [Article.are_search_index_target(:default), Document.are_search_index_target(:default)],
        AreSearch::DumpBody,
        fields: [:title],
    )

MoreLikeThis の例。

    article_index = Article.are_search_index_target(:default)

    body = AreSearch::MoreLikeThis.search(
        [article_index],
        AreSearch::DumpBody,
        article_index,
        fields: [:title, :body],
    )

MoreLikeThis では like 句の _index / _id が "<DUMP_BODY>" というプレースホルダになる。
RawSearch は body を直接渡すため DumpBody には対応しない。
